YOLOv8 辨識 CAPTCHA 驗證碼：從動機到成果

2023/11/26 作品 YOLOv8 CAPTCHA AI 機器學習 驗證碼辨識

引言

在現代網路服務中，CAPTCHA（全自動區分電腦和人類的公開圖靈測試）已成為防止自動化腳本濫用服務的重要手段。然而，對於使用者來說，頻繁輸入 CAPTCHA 驗證碼往往會帶來不便。本文將探討如何利用 YOLOv8 模型來自動辨識 CAPTCHA 驗證碼，從而提升使用者體驗。

 

動機與目的

根據調查，許多台灣人每日使用網路的時間已高達 7 小時以上。網路服務的普及使得人們對網路的依賴日益增加，而 CAPTCHA 驗證碼的輸入過程卻成為了一種不便。因此，我們希望透過 YOLOv8 訓練 AI 模型，讓機器學習辨識驗證碼，藉此提高效率與準確性，進而改善使用者的登入體驗。

架構流程

我們的專案架構分為以下幾個步驟：

  1. 資料準備：蒐集並分類驗證碼，建立資料集。
  2. 模型訓練：使用 YOLOv8 建立並訓練模型，進行模型優化與預測。
  3. 成果展示：在網站上擷取驗證碼，進行辨識並回傳結果。

 

 

資料準備

我們從 Kaggle 和 Roboflow 等平台蒐集了 6746 張驗證碼圖片，並將其分為 62 個類別。資料集進一步分為訓練集（5653 張）和驗證集（1093 張）。

 

模型訓練

我們使用 YOLOv8 進行模型訓練，並調整了多項參數以優化模型性能。訓練過程中，我們使用了以下參數：

▪️epochs: 350 ▪️batch size: 32 ▪️learning rate: 0.0001 ▪️image size: 160 ▪️degrees: 18

技術細節

在選擇 YOLOv8 作為我們的模型架構時，我們主要考慮了其高效能與輕量化的特性。YOLOv8 不僅在目標檢測任務中表現出色，還具有易於部署的優勢，這對於我們這種需要快速迭代的專案來說非常重要。此外，我們在數據準備階段使用了多種數據增強技術，包括旋轉、縮放和平移，以增加數據的多樣性，從而提升模型的泛化能力。

為了進一步評估模型的性能，我們使用了多種評估指標，包括 Precision、Recall、F1 Score 和 mAP（mean Average Precision）。這些指標幫助我們更全面地了解模型在不同類別上的表現，並針對性地進行優化。

 

困難與挑戰

在模型訓練過程中，我們遇到了一些挑戰，例如字母大小寫的區別和驗證碼圖片比例的問題。為了解決這些問題，我們在 Roboflow 上標註時在小寫字母上方留空加以區別，並使用 Selenium 截取網頁畫面以確保驗證碼圖片的比例正確。

1. 因應每台電腦解析度不同

由於不同設備的解析度可能不同，我們需要動態調整瀏覽器窗口的大小，以確保截圖的完整性。以下是相關代碼：

from selenium import webdriver
from selenium.webdriver.common.by import By

# 設置 Chrome 選項，確保瀏覽器在腳本執行完畢後不會自動關閉
option = webdriver.ChromeOptions()
option.add_experimental_option("detach", True)

# 啟動 Chrome 瀏覽器
driver = webdriver.Chrome(options=option)

# 獲取網頁的寬度和高度
width = driver.execute_script("return document.documentElement.scrollWidth")
height = driver.execute_script("return document.documentElement.scrollHeight")

# 根據獲取的寬度和高度設置瀏覽器窗口大小
driver.set_window_size(width, height)

2. 全版截圖

在調整好瀏覽器窗口大小後，我們可以進行全版截圖，並單獨截取驗證碼圖片的區域：

# 截取整個網頁的畫面並保存為文件
driver.get_screenshot_as_file('tcat.png')

# 定位驗證碼元素並截取該元素的圖片
captcha = driver.find_element(By.XPATH, '//*[@id="BookingS1Form_homeCaptcha_passCode"]')
captcha.screenshot("/Users/tingting/Downloads/t_cat/0.jpg")

 

成果展示

經過多次優化，我們的模型在驗證碼辨識上取得了顯著的成果。我們將自動辨識驗證碼的功能應用在自己架設的網站和外部網站（如黑貓、高鐵購票）上，成功實現了省時又便利的目的。

 

 

未來改進方向

未來，我們計劃進一步優化模型的性能，例如通過引入更大的數據集或更複雜的模型架構（如 YOLOv9）來提升辨識準確率。此外，我們也考慮將這項技術應用於其他類型的 CAPTCHA，如圖像選擇或滑動驗證，以擴展其應用場景。

 

結論

透過 YOLOv8 訓練 AI 模型，我們成功實現了驗證碼的自動辨識功能。這不僅解決了生活中的小困擾，也讓我們將課堂學到的理論知識應用於實際情境，達到了學以致用的效果。

然而，我們也意識到這項技術可能帶來的倫理與安全性問題。例如，自動化辨識 CAPTCHA 技術可能被用於惡意目的，如自動化攻擊或數據爬取。因此，我們在開發過程中始終保持對技術合法使用的關注，並建議 CAPTCHA 設計者通過增加驗證碼的複雜度或引入動態變化來應對這種自動化辨識技術。
